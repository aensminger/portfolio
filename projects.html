The product I work on is an IoT Cloud SaaS, and my work ranges from sales demos, to partner integrations (hardware, and embedded software, and enterprise software), to full design and development on behalf of our customers. But there is no IoT without 'Things', and while some of my work is in the cloud, there have been a host of reasons for me to hook up some tangible devices from customers and partners here in my office. Most notably, I wrote a Java based cloud integration plugin for a global leader in remote sensing hardware, which essentially took local sensor data and safely moved it to the cloud, allowing new customers to jump start their IoT application development. System Integrations

One of the most powerful ways to improve functionality in a web application is to integrate with some other web accessible system. I've worked on several of these types of projects - incorporating live weather data into an IoT dashboard, integrating enterprise PLM, CRM and remote connectivity platforms, and pulling remote sensing data from multiple third-party cloud systems to display in a single consolidated environment.

Often the biggest challenge of these types of projects is not the technology required to connect the systems effectively and securely, but the disparate business contexts of each party involved. This means assessing each application's core strengths and limitations to identify the most important points of integration, while developing a unified system for translating between data, process and usage models that are not exactly aligned.
